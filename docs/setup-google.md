# 📊 Настройка Google Sheets + Google Apps Script

Пошаговая инструкция по подключению бэкенда для хранения данных.

---

## Шаг 1. Создать Google Таблицу

1. Перейдите на [sheets.google.com](https://sheets.google.com)
2. Нажмите **+** (Создать пустую таблицу)
3. Назовите таблицу: **Чек-лист магазина**

---

## Шаг 2. Открыть Apps Script

1. В таблице нажмите **Расширения** → **Apps Script**
2. Откроется редактор скриптов
3. Удалите стандартный код `function myFunction() {}`

---

## Шаг 3. Вставить код

1. Откройте файл `google-apps-script/Code.gs` из проекта
2. Скопируйте **весь** код
3. Вставьте его в редактор Apps Script
4. Нажмите **Ctrl+S** (сохранить)
5. Назовите проект: **Чек-лист магазина**

---

## Шаг 4. Инициализировать листы

1. В редакторе Apps Script выберите функцию `initializeSheets` в выпадающем списке сверху
2. Нажмите **▶ Выполнить**
3. При первом запуске Google попросит разрешение:
   - Нажмите **Просмотреть разрешения**
   - Выберите ваш аккаунт Google
   - Нажмите **Дополнительно** → **Перейти к проекту** (внизу)
   - Нажмите **Разрешить**
4. Вернитесь в Google Таблицу — должны появиться 3 листа:
   - `tasks` — выполнение задач
   - `customers_log` — журнал покупателей
   - `config` — настройки

> ⚠️ Можно удалить стандартный «Лист1», если он остался.

---

## Шаг 5. Настроить конфигурацию

Перейдите на лист **config** и заполните:

| key | value |
|---|---|
| telegram_bot_token | *(токен бота — см. [setup-telegram.md](setup-telegram.md))* |
| telegram_chat_id | *(chat_id владельца — см. [setup-telegram.md](setup-telegram.md))* |
| pin_agnesa | 1111 |
| pin_oksana | 2222 |
| pin_owner | 0000 |

> 💡 PIN-коды можно изменить на любые 4 цифры.

---

## Шаг 6. Деплой как Web App

1. В редакторе Apps Script нажмите **Развёртывание** → **Новое развёртывание**
2. Нажмите ⚙️ рядом с «Тип» → выберите **Веб-приложение**
3. Заполните:
   - **Описание**: `Чек-лист v1`
   - **Выполнять как**: **Я** (ваш аккаунт)
   - **У кого есть доступ**: **Все** (Anyone)
4. Нажмите **Развернуть**
5. Скопируйте **URL веб-приложения** — он выглядит так:
   ```
   https://script.google.com/macros/s/AKfycbx.../exec
   ```

> ⚠️ **Важно!** При каждом изменении кода нужно создавать **новое развёртывание**
> или обновлять существующее через **Развёртывание → Управление развёртываниями**.

---

## Шаг 7. Вставить URL в проект

1. Откройте файл `js/config.js` в вашем проекте
2. Найдите строку:
   ```javascript
   API_URL: '',
   ```
3. Вставьте скопированный URL:
   ```javascript
   API_URL: 'https://script.google.com/macros/s/AKfycbx.../exec',
   ```
4. Сохраните файл и загрузите на GitHub

---

## Шаг 8. Настроить триггеры

Триггеры нужны для автоматических напоминаний и проверок дедлайнов.

1. В редакторе Apps Script выберите функцию `setupTriggers`
2. Нажмите **▶ Выполнить**
3. Разрешите доступ, если попросит

Будут созданы триггеры:

| Время | Функция | Описание |
|---|---|---|
| 09:30 | sendMorningReminder930 | Напоминание о утренних задачах |
| 09:50 | sendMorningReminder950 | Последнее напоминание |
| 10:05 | checkMorningDeadline | Проверка дедлайна — кто не выполнил |
| 19:30 | sendEveningReminder1930 | Напоминание о вечерних задачах |
| 19:50 | sendEveningReminder1950 | Последнее напоминание |
| 20:05 | checkEveningDeadline | Проверка дедлайна — кто не выполнил |

> 💡 Проверить триггеры: **Apps Script** → иконка ⏰ (Триггеры) слева.

---

## Шаг 9. Протестировать

1. Выберите функцию `testTelegram` → нажмите **▶ Выполнить**
2. Проверьте, пришло ли сообщение в Telegram
3. Откройте сайт в браузере → войдите как Агнеса → отметьте задачу
4. Проверьте:
   - В Google Таблице появилась запись на листе `tasks`
   - В Telegram пришло уведомление

---

## Обновление кода Apps Script

Если нужно обновить код:

1. Внесите изменения в редакторе Apps Script
2. Нажмите **Развёртывание** → **Управление развёртываниями**
3. Нажмите ✏️ (редактировать) рядом с текущим развёртыванием
4. В поле «Версия» выберите **Новая версия**
5. Нажмите **Развернуть**

> ⚠️ URL Web App останется прежним.

---

## Решение проблем

| Проблема | Решение |
|---|---|
| Ошибка «Нет доступа» | Проверьте: доступ = «Все» в настройках развёртывания |
| Данные не сохраняются | Проверьте URL в config.js, откройте консоль браузера (F12) |
| Триггеры не срабатывают | Проверьте триггеры в Apps Script (иконка ⏰) |
| Telegram не приходит | Проверьте токен и chat_id в листе config |
| Ошибка CORS | Используйте `Content-Type: text/plain` для POST-запросов |
