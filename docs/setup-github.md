# 📦 Публикация на GitHub Pages

Пошаговая инструкция по размещению чек-листа магазина на GitHub Pages.

---

## Шаг 1. Создать аккаунт GitHub

1. Перейдите на [github.com](https://github.com)
2. Нажмите **Sign up** (Регистрация)
3. Введите email, придумайте пароль и имя пользователя
4. Подтвердите email

> 💡 Если аккаунт уже есть — переходите к шагу 2.

---

## Шаг 2. Создать новый репозиторий

1. Нажмите **+** в правом верхнем углу → **New repository**
2. Заполните:
   - **Repository name**: `store-checklist` (или любое другое имя)
   - **Description**: `Чек-лист магазина`
   - **Public** (обязательно — для GitHub Pages)
3. Нажмите **Create repository**

---

## Шаг 3. Загрузить файлы проекта

### Вариант А: Через интерфейс GitHub (простой способ)

1. В созданном репозитории нажмите **uploading an existing file**
2. Перетащите все файлы и папки проекта:
   - `index.html`
   - `employee.html`
   - `admin.html`
   - `manifest.json`
   - `sw.js`
   - папку `css/`
   - папку `js/`
   - папку `icons/`
3. Внизу нажмите **Commit changes**

> ⚠️ GitHub не позволяет загружать папки через drag & drop напрямую.
> Загрузите файлы внутри папок отдельно, предварительно создав папки.

### Вариант Б: Через Git (если знаете как)

```bash
git init
git add .
git commit -m "Первый коммит — чек-лист магазина"
git branch -M main
git remote add origin https://github.com/ВАШ_ЛОГИН/store-checklist.git
git push -u origin main
```

---

## Шаг 4. Включить GitHub Pages

1. Перейдите в репозиторий на GitHub
2. Нажмите **Settings** (Настройки)
3. В левом меню выберите **Pages**
4. В разделе **Source**:
   - Branch: **main**
   - Folder: **/ (root)**
5. Нажмите **Save**

---

## Шаг 5. Дождаться публикации

- GitHub начнёт публикацию (обычно 1–3 минуты)
- После этого появится ссылка вида:
  ```
  https://ВАШ_ЛОГИН.github.io/store-checklist/
  ```
- Перейдите по ссылке — вы должны увидеть страницу входа

---

## Шаг 6. Настроить API URL

1. После деплоя Google Apps Script (см. [setup-google.md](setup-google.md)) скопируйте URL Web App
2. Откройте файл `js/config.js` в репозитории GitHub
3. Нажмите кнопку ✏️ (редактирование)
4. Замените пустую строку `API_URL: ''` на ваш URL:
   ```javascript
   API_URL: 'https://script.google.com/macros/s/ВАШ_КЛЮЧ/exec',
   ```
5. Нажмите **Commit changes**
6. Подождите 1–2 минуты пока обновится GitHub Pages

---

## Шаг 7. Поделиться ссылкой

Отправьте сотрудникам ссылку на ваш сайт:

```
https://ВАШ_ЛОГИН.github.io/store-checklist/
```

Инструкция для сотрудников — см. [guide-employee.md](guide-employee.md)

---

## Обновление проекта

Если нужно обновить файлы:

1. Откройте файл в репозитории GitHub
2. Нажмите ✏️ для редактирования
3. Внесите изменения
4. Нажмите **Commit changes**
5. GitHub Pages обновится автоматически через 1–2 минуты

---

## Решение проблем

| Проблема | Решение |
|---|---|
| Сайт не открывается | Проверьте что Pages включен (Settings → Pages) |
| Страница 404 | Убедитесь что `index.html` в корне репозитория |
| Старая версия | Очистите кэш браузера (Ctrl+Shift+R) |
| Не работает PWA | Проверьте что manifest.json загружен |
